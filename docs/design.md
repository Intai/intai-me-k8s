# Infrastructure-as-Code: Packer + KVM + Kubespray + Helm for Kubernetes on EC2

## Context

Deploy the intai.me website (nginx serving a CV page) as a Kubernetes pod running inside KVM virtual machines. Each host (AWS EC2 c8i.large or physical Linux server) runs one KVM VM, and Kubespray joins all VMs into a single K8s cluster. Packer builds a QEMU/KVM image with the OS pre-installed. Ansible configures hosts, provisions VMs, sets up WireGuard tunnels for inter-host communication, and runs Kubespray. Helm deploys Traefik, cert-manager, and the nginx site. Terraform provisions AWS infrastructure only. The design is inventory-driven — adding a server requires only incrementing `SERVER_COUNT` (AWS) or editing `hosts.yml` (physical).

## Traffic Flow

```
Internet
  │
  ▼
Route 53 (weighted routing + health checks)
  ├─ healthy? → EIP A (Host A) → DNAT → VM node1 → Traefik → site pods
  └─ healthy? → EIP B (Host B) → DNAT → VM node2 → Traefik → site pods

node1 (.122.10) ◄── WireGuard tunnel (between hosts) ──► node2 (.122.11)
                    (etcd, K8s API, pod network)
```

Each host forwards ports 80/443/6443 via iptables DNAT to its local VM. Traefik runs as a DaemonSet with `hostNetwork: true` so each node serves traffic independently. DNS routes users to healthy hosts; Route 53 health checks automatically remove unhealthy hosts from DNS responses.

cert-manager handles TLS via HTTP-01 challenge. This works across multiple nodes because the ACME challenge pod is reachable from any Traefik instance via Kubernetes pod networking (routed over WireGuard).

## Implementation Plan

### Phase 1: Scaffolding & Web Content

- Create `pyproject.toml` with dependencies: `ansible`, `boto3`
- Create `.env.example`:
  ```
  AWS_REGION=ap-southeast-2
  DOMAIN_NAME=intai.me
  CERTBOT_EMAIL=intai.hg@gmail.com
  INSTANCE_TYPE=c8i.large
  PROJECT_NAME=intai-me-k8s
  ENVIRONMENT=production
  VM_CPUS=2
  VM_MEMORY_MB=3072
  VM_DISK_GB=30
  SERVER_COUNT=1
  ```
- Create `.gitignore` (tfstate, .terraform/, .env, .venv/, *.qcow2, output-*)
- Create `helm/nginx-site/files/index.html` — minimal HTML with full-viewport `<iframe>` rendering `cv.pdf`
- Place `cv.pdf` directly in `helm/nginx-site/files/`
- Create `ansible/requirements.yml` — Kubespray as Ansible collection pinned to `v2.30.0`

### Phase 2: Packer — KVM Base Image (`packer/`)

Packer uses the QEMU builder to produce a hardened Ubuntu qcow2 image. Cloud-init installs the OS with a temporary `packer` user (password auth) so Packer can SSH in during the build. A shell provisioner then locks the image down before finalization. Kubespray handles all K8s prerequisites and installation post-deploy.

- **k8s-node.pkr.hcl**:
  - `source "qemu"` block:
    - `iso_url`: Ubuntu 24.04 Server ISO
    - `iso_checksum`: SHA256 from Ubuntu releases
    - `disk_size`: `20G`
    - `format`: `qcow2`
    - `accelerator`: `kvm` (macOS: `hvf`; CI/Linux: `kvm`)
    - `http_directory`: `http/` (serves cloud-init)
    - `boot_command`: autoinstall kernel params pointing to cloud-init HTTP server
    - `ssh_username`/`ssh_password`: hardcoded `packer`/`packer` for build-time access only
    - `shutdown_command`: `sudo shutdown -P now`
  - Hardening provisioner — Ansible playbook (`playbook-image.yml`) with `image_hardening` role:
    1. Installs and configures fail2ban with SSH jail (5 retries / 10 min window / 1 hour ban)
    2. Disables SSH password authentication (`PasswordAuthentication no`)
    3. Locks the packer user's password
    - `NOPASSWD` sudo rule is kept — Kubespray needs it. Removed post-deploy by `playbook-k8s.yml`
    - SSH keys are injected at deploy time by Ansible's `virt-customize`, not baked into the image
  - Output: `output-k8s-node/k8s-node.qcow2`

- **variables.pkr.hcl**: `ubuntu_iso_url`, `ubuntu_iso_checksum`, `disk_size`

- **http/user-data**: cloud-init autoinstall config — sets up SSH user with password auth for Packer build-time access (disabled in final image by provisioner)

### Phase 3: Ansible — Host Configuration (`ansible/`)

Ansible configures any Linux host (EC2 or physical) to run a KVM VM. The playbook is structured as multiple plays to support WireGuard key exchange across hosts.

#### Role: `kvm_host`
1. Install packages: `qemu-kvm`, `libvirt-daemon-system`, `virtinst`, `libguestfs-tools`, `iptables-persistent`, `fail2ban`
2. Configure and enable fail2ban with SSH jail (5 retries / 10 min window / 1 hour ban)
3. Enable and start `libvirtd` service
4. Detect CPU vendor (Intel/AMD), enable nested KVM if not already active
5. Ensure libvirt default network (virbr0, 192.168.122.0/24) is active

#### Role: `vm_provision`
1. Copy Packer-built qcow2 image to `/var/lib/libvirt/images/<node_name>.qcow2`
2. Use `virt-customize` to inject SSH key and set static IP (from `vm_ip` host var) via cloud-init/netplan
3. Define VM from `vm-domain.xml.j2` template via `virsh define`:
   - VM name and disk path derived from `node_name` host var
   - vCPUs: `{{ vm_cpus }}` (default: 1)
   - Memory: `{{ vm_memory_mb }}` MB (default: 2048)
   - Network: libvirt default (virbr0)
4. Start VM via `virsh start`, wait for SSH to become available
5. Add VM to in-memory Ansible inventory using `node_name`

#### Role: `wireguard`
Sets up WireGuard tunnels between hosts for inter-VM K8s communication. Runs in two phases:
1. **generate_keys** — install `wireguard-tools`, generate key pair, register public key as Ansible fact
2. **configure** — template `wg0.conf` with all peers (full mesh), enable `wg-quick@wg0`. PostUp routes each peer's `vm_ip` through the WireGuard interface.

The two-phase approach ensures all hosts' public keys are available before any host templates its config.

#### Role: `port_forward`
1. Enable IP forwarding: `sysctl net.ipv4.ip_forward=1`
2. iptables DNAT rules for ports 80/443/6443 from host public interface to `vm_ip` (from host inventory)
3. MASQUERADE and FORWARD rules for virbr0
4. Persist rules via `netfilter-persistent save`

#### Playbook: `playbook-host.yml`
```yaml
- hosts: hosts       # Play 1: KVM + VM provisioning
  roles: [kvm_host, vm_provision]

- hosts: hosts       # Play 2: Generate WireGuard keys
  tasks: [include_role wireguard/generate_keys]

- hosts: hosts       # Play 3: Configure WireGuard mesh
  tasks: [include_role wireguard/configure]

- hosts: hosts       # Play 4: Port forwarding
  roles: [port_forward]
```

### Phase 4: Kubespray — K8s Install & Cluster Formation (post-deploy)

After all hosts are provisioned and VMs are running, Kubespray runs once to install K8s and form the cluster across all VMs.

#### Auto-generated inventory

`k8s-cluster.yml` is auto-generated from `hosts.yml` by the `generate_cluster` role. Each host's `node_name`, `vm_ip`, and `ansible_host` are used to build node entries with per-node SSH ProxyJump through the host's public IP. All nodes are placed in `kube_control_plane`, `kube_node`, and `etcd` groups.

#### Kubespray vars:
- `container_manager: containerd`
- `kube_network_plugin: calico` (or flannel)
- `auto_renew_certificates: true`

#### Scaling to multiple servers:
1. Increment `SERVER_COUNT` and run `make tf-apply` (AWS) — Terraform generates updated `hosts.yml`
2. Or add host entry to `hosts.yml` manually (physical servers)
3. Run `make kvm-setup` — provisions VM on new host, sets up WireGuard mesh
4. Run `make k8s-cluster` — auto-generates `k8s-cluster.yml`, Kubespray joins new node

#### Dependencies (deployed via Helmfile):
- **Traefik**: Helm chart deployed as DaemonSet with `hostNetwork: true` so each node binds directly to VM ports 80/443
- **cert-manager**: Helm chart + CRDs for automatic Let's Encrypt certificates (HTTP-01)

### Phase 6: Helm Deployment — Helmfile

All Helm releases are declared in `helmfile.yaml.gotmpl` at the repo root.

#### `helmfile.yaml.gotmpl`
```yaml
repositories:
  - name: jetstack
    url: https://charts.jetstack.io
  - name: traefik
    url: https://traefik.github.io/charts

releases:
  - name: cert-manager
    namespace: cert-manager
    chart: jetstack/cert-manager
    set:
      - name: crds.enabled
        value: "true"

  - name: traefik
    namespace: traefik
    chart: traefik/traefik
    needs: [cert-manager/cert-manager]
    values:
      - deployment:
          kind: DaemonSet
        hostNetwork: true
        dnsPolicy: ClusterFirstWithHostNet

  - name: nginx-site
    chart: ./helm/nginx-site
    needs: [traefik/traefik]
    values:
      - domain: {{ requiredEnv "DOMAIN_NAME" }}
        certEmail: {{ requiredEnv "CERTBOT_EMAIL" }}
        replicaCount: {{ env "SERVER_COUNT" | default "1" }}
```

### Phase 7: Terraform Modules — AWS Infrastructure Only (`terraform/`)

Terraform provisions the AWS environment and generates `ansible/inventory/hosts.yml`.

#### VPC Module (`modules/vpc/`)
- VPC `10.0.0.0/16` with DNS support/hostnames enabled
- 2 public subnets (`10.0.1.0/24`, `10.0.2.0/24`) across 2 AZs
- Internet gateway + public route table
- Outputs: `vpc_id`, `public_subnet_ids`

#### Security Module (`modules/security/`)
- Security group:
  - Ingress 80 from `0.0.0.0/0` (HTTP — Let's Encrypt + redirect)
  - Ingress 443 from `0.0.0.0/0` (HTTPS)
  - Ingress 6443 from `0.0.0.0/0` (K8s API — authenticated via client certificate)
  - Ingress 22 from `0.0.0.0/0` (SSH — key-based auth)
  - Ingress 51820/udp from `0.0.0.0/0` (WireGuard)
  - Egress all
- Outputs: `instance_sg_id`

#### Compute Module (`modules/compute/`)
- `data "aws_ami"` lookup for latest Ubuntu 24.04 official AMI
- `aws_instance` with `count = var.server_count`:
  - Distributed across subnets/AZs for availability
  - Encrypted gp3 root volume, nested virtualization enabled, IMDSv2 enforced
- `aws_eip` per instance
- Outputs: `instance_ids`, `public_ips`, `elastic_ips` (all lists)

#### DNS Module (`modules/dns/`)
- Route 53 hosted zone
- `aws_route53_health_check` per EIP (HTTPS on port 443)
- `aws_route53_record` with weighted routing policy per EIP (scales to N servers)
- Unhealthy hosts automatically removed from DNS responses
- Same pattern for `www.` records

#### Root Module
- **main.tf**: Compose modules + `local_file` resource to generate `hosts.yml` from compute outputs
- **templates/hosts.yml.tftpl**: Terraform template for Ansible inventory
- **providers.tf**: AWS + local providers
- **variables.tf**: `aws_region`, `project_name`, `environment`, `domain_name`, `instance_type`, `server_count`
- **outputs.tf**: Surface lists of instance IDs, EIPs, VPC ID

## Resource Allocation

| Resource | EC2 Host (c8i.large) | KVM VM       |
|----------|----------------------|--------------|
| vCPUs    | 2                    | 2            |
| Memory   | 4 GB                 | 3 GB         |
| Disk     | 45 GB gp3            | 30 GB qcow2  |
| Network  | Public (EIP)         | 192.168.122.x (virbr0) |

Each host runs one VM. With `SERVER_COUNT=2`, two separate c8i.large instances are provisioned.

## Key Design Decisions

1. **No AMI — portable to physical servers**: Packer builds a qcow2 KVM image, not an AWS AMI. The same image runs on EC2 or any Linux server with KVM. Terraform is AWS-only; Ansible handles everything else.
2. **Kubespray does everything post-deploy**: Packer builds a clean Ubuntu image with OS prerequisites only. Kubespray handles full K8s install + cluster formation in one run — the standard workflow. Same image works for single-node or multi-node, just change the inventory.
3. **KVM on c8i.large (nested virtualization)**: 8th-gen Intel EC2 instances support nested virtualization via `cpu_options { nested_virtualization = "enabled" }`, exposing the `vmx` CPU flag to enable KVM inside EC2 without expensive bare-metal instances.
4. **Traefik replaces ingress-nginx**: ingress-nginx was retired in March 2026. Traefik runs as a DaemonSet with `hostNetwork: true` — each node independently serves traffic. No need for kube-vip or MetalLB since nodes are on separate L2 networks.
5. **cert-manager with HTTP-01**: cert-manager handles Let's Encrypt certificates via HTTP-01 challenge. Works across multiple nodes because the ACME challenge pod is reachable from any Traefik instance via Kubernetes pod networking (routed over WireGuard).
6. **WireGuard for inter-host communication**: VMs on different hosts communicate via WireGuard tunnels between hosts. Each VM gets a unique IP (192.168.122.10, .11, ...) and the host routes peer VM IPs through the WireGuard interface.
7. **Route 53 weighted routing + health checks**: DNS-based traffic failover. Each server gets equal weight; unhealthy servers are automatically removed from DNS responses. Failover takes 30-90 seconds (DNS TTL + health check detection).
8. **Inventory-driven scaling**: On AWS, increment `SERVER_COUNT` and run `make tf-apply` — Terraform generates `hosts.yml` automatically. For physical servers, edit `hosts.yml` manually. `k8s-cluster.yml` is auto-generated from `hosts.yml`. No code changes to scale.
9. **Auto-renewing K8s certificates**: Kubespray's `auto_renew_certificates: true` sets up a systemd timer to renew certs before they expire (default 1 year). Re-fetch kubeconfig annually via `make k8s-config`.
10. **Kubespray as Ansible Collection**: Installed via `ansible-galaxy` from `ansible/requirements.yml`, pinned to `v2.30.0`. Cleaner than a git submodule — no 50MB+ directory in the repo, standard Ansible dependency management.
11. **IMDSv2 enforced**: Prevents SSRF credential theft on the EC2 host.
12. **SSH for remote access**: Key-based auth only. Packer's hardening provisioner disables password auth and locks the build-time password. SSH keys are injected at deploy time by Ansible's `virt-customize`.

## Adding a Server

### AWS
1. Set `SERVER_COUNT=2` in `.env`
2. `make tf-apply` — creates 2nd EC2 + EIP + health check + DNS record + updates `hosts.yml`
3. `make kvm-setup` — provisions VM, sets up WireGuard mesh, port forwarding
4. `make k8s-cluster` — auto-generates `k8s-cluster.yml`, Kubespray joins node2
5. `make helm-apply` — Traefik DaemonSet auto-schedules on node2

### Physical servers
1. Edit `hosts.yml` — add server2 entry with IP, vm_ip, wg_ip, node_name
2. `make kvm-setup` → `make k8s-cluster` → `make helm-apply`

## Verification

1. `make validate` — runs `packer validate` and `terraform validate`
2. `make image` — builds qcow2 image
3. `make tf-init && make tf-plan` — review Terraform plan
4. `make aws-deploy` — full AWS deployment
5. Verify via SSH:
   - `virsh list` — VM is running
   - `wg show` — WireGuard tunnel is up (multi-server)
   - `kubectl get nodes` — all nodes Ready
   - `kubectl get pods -A` — Traefik, cert-manager, nginx pods running
6. `curl -I https://intai.me` — HTTPS with valid cert
7. `curl -I http://intai.me` — redirects to HTTPS
8. `curl -I https://www.intai.me` — redirects to non-www
9. Kill one node — DNS health check fails, traffic shifts to remaining server
